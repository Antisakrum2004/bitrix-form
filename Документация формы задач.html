Bitrix24 Task Creator
Полная документация

Одностраничное приложение для создания задач в Bitrix24 через REST API. Чистый HTML + CSS + JS без зависимостей. Тёмная тема с glass-морфизмом и анимированным огоньком важности.

Zero Dependencies Single HTML Bitrix24 REST Dark Theme LocalStorage
01 — Возможности

Что умеет форма

  • Создание задач через tasks.task.add с полной параметризацией
  • 7 секций описания — цель, что сделать, зависимости, критерии приёмки, материалы (ссылки), материалы (файлы), контакт
  • Тег-инпут для проектов с чипами и управлением через клавиатуру
  • Огонёк важности — 4-слойная SVG-иконка с анимацией зажигания/тушения, переключает PRIORITY 0↔2
  • Планирование сроков — ввод часов, автокалькуляция рабочих дней и END_DATE_PLAN
  • Загрузка файлов — drag & drop или выбор, Base64-кодирование, прикрепление через tasks.task.files.attach
  • Предпросмотр описания — живая сборка из заполненных секций
  • Настройки вебхука — выдвижная панель, сохранение в LocalStorage
  • Автоопределение ID — кнопка «Узнать мой ID» через profile.json
  • Авто-размер textarea — поля растут по содержимому
  • Индикаторы секций — SVG-иконки меняют цвет при заполнении
  • Валидация — обязательные поля с inline-ошибками и toast-уведомлениями
  • Адаптивность — на мобильных sidebar скрывается, сетка перестраивается

Почему без фреймворков

Форма — единственная страница. Реактивность не нужна, состояние минимально. Vanilla JS даёт полный контроль, нулевой бандл, мгновенную загрузку.

LЛогика переключения

JS — toggleFlame()
let flameOn = false;

function toggleFlame() {
  const icon = document.getElementById('flameIcon');
  // Блокировка на время анимации
  if (icon.classList.contains('igniting') ||
      icon.classList.contains('extinguishing')) return;

  if (!flameOn) {
    flameOn = true;
    icon.classList.add('lit', 'igniting');
    setTimeout(() => icon.classList.remove('igniting'), 460);
  } else {
    flameOn = false;
    icon.classList.remove('lit');
    icon.classList.add('extinguishing');
    setTimeout(() => icon.classList.remove('extinguishing'), 320);
  }
}
Связь с API

Когда flameOn === true, в payload отправляется PRIORITY: 2 (высокий). Когда false — PRIORITY: 0 (низкий). Никаких промежуточных значений.

11 — Тег-инпут

Система тегов

Кастомный инпут с чипами внутри flex-контейнера. Без библиотек — чистый DOM.

KКлавиатурное управление

КлавишаДействие
EnterДобавить тег из инпута
ЗапятаяТо же, что Enter
Backspace (пустой инпут)Удалить последний тег

RРендеринг

JS — renderTags()
function renderTags() {
  const wrap  = document.getElementById('tagsWrap');
  const input = document.getElementById('tagInput');
  wrap.innerHTML = '';

  window.tags.forEach((tag, i) => {
    const chip = document.createElement('span');
    chip.className = 'tag-chip';
    chip.innerHTML = `${escHtml(tag)}` +
      ` <span class="tag-chip-remove"` +
      ` onclick="removeTag(${i})">×</span>`;
    wrap.appendChild(chip);
  });

  wrap.appendChild(input);  // инпут всегда в конце
}
Дедупликация

Функция addTag() проверяет !window.tags.includes(t) перед добавлением. Пустые строки и дубли игнорируются.

12 — Файлы

Загрузка файлов

Двухэтапный процесс: сначала создание задачи, потом прикрепление файлов по полученному ID.

1Выбор файлов

Поддерживаются два способа: drag & drop на зону и клик для вызова системного диалога. Обработчики:

JS — обработчики
// Клик — скрытый input[type=file]
fileInput.addEventListener('change', e => {
  handleFiles(Array.from(e.target.files));
  fileInput.value = '';  // сброс для повторного выбора
});

// Drag & drop
dropZone.addEventListener('dragover', e => {
  e.preventDefault();
  dropZone.classList.add('drag-over');
});
dropZone.addEventListener('drop', e => {
  e.preventDefault();
  dropZone.classList.remove('drag-over');
  handleFiles(Array.from(e.dataTransfer.files));
});

2Base64-кодирование

JS — toB64()
function toB64(file) {
  return new Promise((resolve, reject) => {
    const reader = new FileReader();
    reader.onload = () =>
      resolve(reader.result.split(',')[1]);
    reader.onerror = reject;
    reader.readAsDataURL(file);
  });
}
Ограничение Base64

Base64 увеличивает размер файла на ~33%. Для файлов больше 5 МБ может быть медленно. Bitrix24 REST через вебхук имеет свои лимиты — обычно до 5 МБ на файл.

3Прогресс-бар

Линейный прогресс через CSS transition: width 0.3s ease. Обновляется после каждого файла в цикле. Показывается через класс .show, прячется после завершения.

13 — Autosize

Автоматический размер textarea

Классический паттерн без библиотек. Сброс высоты → чтение scrollHeight → установка.

JS — autoGrow()
function autoGrow(el) {
  el.style.height = 'auto';     // сброс → минимальная
  el.style.height = el.scrollHeight + 'px';  // по содержимому
}

Вызывается на input и paste (paste через setTimeout для ожидания вставки). Начальная высота через min-height: 38px и rows="1".

function saveWebhook() {

18 — Профиль

profile.json — определение ID

Кнопка «Узнать мой ID» вызывает endpoint профиля и извлекает ID текущего пользователя вебхука.

HTTP
GET {WEBHOOK_URL}profile.json
JS — fetchMyId()
async function fetchMyId() {
  const hook = getWebhook();
  const res = await fetch(hook + 'profile.json');
  const data = await res.json();

  if (data.result?.ID) {
    document.getElementById('responsibleId').value = data.result.ID;
    localStorage.setItem('bx_responsible_id', data.result.ID);
    showToast('success', 'ID получен: ' + data.result.ID,
      data.result.NAME + ' ' + (data.result.LAST_NAME || ''));
  }
}

———Теги, описание, файлы НЕ сохраняются
Почему не всё в localStorage

Теги, описание и файлы — одноразовые данные. Задача создана — форма очищается. Хранить их бессмысленно и может запутать пользователя при следующем открытии.

20 — Безопасность

Соображения безопасности

  • Вебхук в localStorage — доступен через DevTools любой вкладки на этом домене. Для продакшена нужен серверный прокси.
  • XSS в описании — текст из textarea не рендерится как HTML (textContent в превью), но отправляется в Bitrix24 как есть. Bitrix24 сам экранирует при отображении.
  • Base64 файлов — отправляется в открытом виде. Без HTTPS перехват возможен.
  • CORS — Bitrix24 разрешает кросс-доменные запросы к вебхукам. Это официальная функция, не баг.
  • Нет CSRF-защиты — вебхук-токен выполняет роль аутентификации. Если утечёт — полный доступ по правам вебхука.
Для продакшена

Сделайте серверный прокси (Node/Python/PHP), который хранит вебхук в env-переменных. Фронтенд отправляет данные на ваш сервер, а сервер — в Bitrix24. Это исключает утечку токена из браузера.

22 — CORS

Проблема CORS и её решение

Не открывай файл:// напрямую

Если открыть file:///path/index.html — fetch к Bitrix24 упадёт с CORS-ошибкой. Браузер блокирует cross-origin запросы из файловой системы.

Форма определяет TypeError и показывает понятное сообщение:

JS — обработка CORS
} catch(err) {
  if (err.name === 'TypeError')
    showToast('error', 'CORS ошибка',
      'Открой через Live Server или python -m http.server');
  else
    showToast('error', 'Сетевая ошибка', err.message);
}
Почему Bitrix24 пропускает

Bitrix24 устанавливает Access-Control-Allow-Origin: * для REST-вебхуков. Запросы с http://localhost:* или http://127.0.0.1:* проходят. С file:// — нет, это особенность браузера.
