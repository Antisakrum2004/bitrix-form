## Кратко: что добавилось в v7.28

| Возможность | Где работает | Ценность |
|---|---|---|
| **Автоподстановка аналогичных задач + дерево** в описание при создании | Форма задачи (index.html) | Экономит исполнителю 5-10 минут на поиск контекста — сразу видит 3 похожих + их 2-х соседей |

## Что сделано — подробно

### 1. Автоподстановка аналогичных задач в DESCRIPTION (1.2 + 2.7)

**Проблема:** Исполнитель получает задачу без контекста — нужно самому идти в поиск, вбивать похожие слова, отвлекаться.

**Что сделали:** При создании задачи через «быструю задачу» — фронт автоматически:
1. Запрашивает top-3 похожих задач через `/api/ai-similar` (порог sim ≥ 0.65, fallback top-3 из sim ≥ 0.4)
2. Для каждого из top-3 — ещё один запрос на его top-2 детей (sim ≥ 0.5, исключая текущего)
3. Собирает BBCode-блок «⚠ Подсказка ИИ — аналогичные задачи»
4. Дописывает в конец DESCRIPTION перед `tasks.task.add`

**Формат блока:**
```
[b]⚠ Подсказка ИИ — аналогичные задачи (проверьте релевантность):[/b]

[b][url=...]#6258[/url][/b] (52% · Отложена · Елена) — Разграничить права в ут 11.5 по пользователям
    └ [url=...]#6434[/url] (65% · Отложена) — Настроить доступы пользователей
[b][url=...]#7260[/url][/b] (47% · Отложена · Ольга) — [IMP] Продавцы видят валовые друг друга
[b][url=...]#6724[/url][/b] (45% · Отложена · Саша) — Доработка в подборе номенклатуры
```

Bitrix24 рендерит это в кликабельные ссылки прямо в задаче.

**Тест:** Создана задача #7990 с заголовком «[ТЕСТ v7.28] Настроить права доступа пользователей к складским остаткам» — блок аналогичных задач на месте, BBCode валидный, 3 родительские задачи + 1 дочерняя.

### 2. Страница поиска типа Яндекс (1.3)

**Что сделали:** Новый статический файл `search.html` на GitHub Pages:
- URL: https://antisakrum2004.github.io/bitrix-form/search.html
- Стартовый экран: большой заголовок «База знаний», инпут с placeholder «опишите задачу: минус резерв, выгрузка в excel, права доступа…», 6 чипсов-примеров
- Фильтры сверху (еле видимые, opacity 0.55 → 1 при hover): статус, сортировка, порог, лимит
- По умолчанию: показывать по релевантности, порог 40%, лимит 20
- Результат: карточка с ID + %sim + кликабельным заголовком (= ссылка в Bitrix24) + 2-строчным превью описания + мета-строкой (статус · исполнитель · проект · дата)

**Бэкенд-изменения:**
- `/api/ai-similar` теперь возвращает `description` (обрезанный до 500 символов) и `created_at`
- RPC `search_similar_tasks` обновлён до v7.28 — добавлен `created_at` в RETURNS TABLE
- SQL применён к Supabase через Management API

### 3. Детектор повторяющихся проблем (2.2)

**Что сделали:**
1. Новый SQL-функция `cluster_repeated_tasks(sim_threshold, min_cluster_size, max_clusters)` — жадная star-кластеризация на стороне БД
2. Новая таблица `task_clusters` — сохраняет результаты для истории
3. Скрипт `scripts/cluster-analytics.mjs` — вызывает RPC, upsert в `task_clusters`, печатает алерты
4. Cron: каждое воскресенье 06:00 МСК через `.github/workflows/weekly-analytics.yml`

**Тестовый прогон (текущее состояние базы):**
- 9 кластеров с ≥5 задачами
- 5 алертов за последние 30 дней (≥5 задач)
- Реальные повторяющиеся проблемы:
  - `min-5238` — 157 задач «Просроченные задачи» (ежедневные уведомления — шумовой кластер)
  - `min-5618` — **5 задач «Проверить загрузку остатков от поставщиков»** за 4 месяца — кандидат на root cause
  - Лекции и обеды — технический шум

**Ценность:** Видим тренды, до которых раньше нельзя было докопаться — что в команде одна и та же проблема возникает регулярно, но каждый раз её чинят точечно.

### 4. Библиотека решений (2.3)

**Что сделали:**
1. Расширена таблица `tasks`: 3 новые колонки — `solution_text`, `solution_embedding` (VECTOR 1536), `solution_indexed_at`
2. Скрипт `scripts/sync-solutions.mjs`:
   - Тянуть комментарии через `task.commentitem.getlist`
   - Фильтровать ботов (id=154) и EOD-фразы
   - Брать последний осмысленный (>20 символов) комментарий как «решение»
   - Получать embedding решения, сохранять в Supabase
3. Cron: еженедельно, после кластеризации

**Текущий статус:** У задач в Bitrix24 комментарии практически отсутствуют (процесс использует EOD-бота вместо комментариев). Скрипт работает корректно, но `solution_text` = NULL для большинства задач. Когда комментарии появятся — индекс будет готов.

### 5. Авто-тегирование задач (2.6)

**Что сделали:**
1. Колонка `tags TEXT[]` добавлена в таблицу `tasks` + gin-индекс
2. 8 предопределённых тегов: бухгалтерия, интеграции, остатки, права, отчёты, эдо, обучение, инфраструктура
3. Скрипт `scripts/index-tags.mjs`:
   - LLM (DeepSeek → Gemini → GPT-4o-mini fallback chain)
   - System prompt: «Верни JSON массив 0-3 тегов из списка...»
   - Валидация: только теги из разрешённого списка
   - Максимум 3 тега на задачу
4. Cron: еженедельно, индексирует новые задачи (где `tags IS NULL` или пустой массив)
5. Для старых задач — `node scripts/index-tags.mjs --force` реиндексирует всё

**Тест на 5 задачах:**
- 2 получили теги (обучение, инфраструктура)
- 3 без тегов (лекции с generic titles — LLM не смог классифицировать)

## Как использовать новые возможности

### Создать задачу с автоподсказками
1. Открыть https://antisakrum2004.github.io/bitrix-form/
2. Нажать AI → описать задачу → нажать «Создать»
3. В DESCRIPTION автоматически появится блок «⚠ Подсказка ИИ — аналогичные задачи» с топ-3 + деревом

### Поиск по базе знаний
1. Открыть https://antisakrum2004.github.io/bitrix-form/search.html
2. Ввести запрос (например «минус резерв» или «права доступа»)
3. Подкорректировать фильтры сверху при необходимости
4. Клик по карточке → откроется задача в Bitrix24

### Посмотреть кластеры повторов
1. Actions tab → weekly-analytics → Run workflow
2. Или: `node scripts/cluster-analytics.mjs` локально
3. В логе — топ-10 кластеров + алерты (≥5 задач за 30 дней)
4. Сохранено в таблице `task_clusters` (можно запросить через Supabase Dashboard)

### Реиндексировать теги вручную
```bash
node scripts/index-tags.mjs              # только новые
node scripts/index-tags.mjs --force      # все
node scripts/index-tags.mjs --limit 50   # 50 штук для теста
```

### Реиндексировать решения
```bash
node scripts/sync-solutions.mjs          # только новые (без solution_text)
node scripts/sync-solutions.mjs --force   # все
```
