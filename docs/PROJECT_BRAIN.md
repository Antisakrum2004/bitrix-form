# PROJECT_BRAIN

## 1. Project Overview

- **Что это:** Одностраничное web-приложение для создания задач в Bitrix24 через REST API (входящий вебхук)
- **Задача:** Упростить и стандартизировать постановку задач в команде — выбрать разработчика, постановщика, проект, дедлайн, заполнить структурированное описание, прикрепить файлы, добавить EOD-комментарий от бота — всё в одном клике
- **Стек технологий:**
  - HTML5 (один файл `index.html`, ~1682 строки)
  - CSS3 (Custom Properties, Grid, Flexbox, backdrop-filter, keyframes)
  - Vanilla JS (ES6+, async/await, Fetch API, FileReader API, XHR)
  - Google Fonts: JetBrains Mono + Unbounded
  - Inline SVG иконки (без библиотек)
  - LocalStorage для конфигурации и черновиков
  - Bitrix24 REST API (два входящих вебхука: пользователь + бот)
- **Репозиторий:** `https://github.com/Antisakrum2004/bitrix-form`
- **Деплой:** GitHub Pages — `https://antisakrum2004.github.io/bitrix-form/`
- **Текущая версия:** v7.12

## 2. Architecture

### Общая схема

```
┌────────────────────────────────────────────────────────┐
│  Браузер (index.html)                                  │
│  ┌──────────┐  ┌──────────┐  ┌─────────────────────┐  │
│  │  UI Form │→ │  JS App  │→ │ Bitrix24 REST API   │  │
│  │ (CSS/    │  │ (Vanilla │  │ ┌─────┐ ┌────────┐  │  │
│  │  HTML)   │  │  async)  │  │ │Hook │ │EOD Bot │  │  │
│  │          │  │          │  │ │ 116 │ │ Hook   │  │  │
│  │          │  │          │  │ │(user│ │ 154    │  │  │
│  │          │  │          │  │ │ +ev)│ │(bot)   │  │  │
│  └──────────┘  └──────────┘  │ └─────┘ └────────┘  │  │
│                     ↕        └─────────────────────┘  │
│              LocalStorage                               │
│       (вебхук, черновик, история задач)                │
└────────────────────────────────────────────────────────┘
```

- **Backend:** Отсутствует. Прямые вызовы Bitrix24 REST API из браузера
- **Frontend:** Единый HTML-файл со встроенными CSS и JS
- **Интеграция:** Bitrix24 REST API через два входящих вебхука:
  - **Хук пользователя (116):** создание задач, файлы, наблюдатели — все основные операции
  - **Хук бота (154):** только EOD-комментарии — чтобы комментарии приходили от имени «Айти Лаб Bot»

### Ключевые зависимости

- `Google Fonts CDN` — шрифты (JetBrains Mono, Unbounded)
- `Bitrix24 REST API` — все данные и операции
- Внешних npm-пакетов нет

### Как идут данные

1. Пользователь заполняет форму → JS собирает payload
2. `POST {user-hook}/tasks.task.add` → создание задачи, получение `task.id` (включая TIME_ESTIMATE если указана оценка)
3. `POST {user-hook}/task.observers.add` (до 4 вариантов эндпоинта) → добавление наблюдателя
4. Если EOD-галка стоит: `POST {user-hook}/addObserver` с userId=154 → бот становится наблюдателем
5. Если EOD-галка стоит: `POST {bot-hook}/task.commentitem.add` → EOD-комментарий от бота
6. `GET {user-hook}/disk.folder.uploadfile?id=3` → получение upload URL
7. Загрузка файла на upload URL (FormData метод)
8. `POST {user-hook}/tasks.task.update` с `UF_TASK_WEBDAV_FILES: ['n'+diskId]` → привязка файла к задаче
9. Сброс формы, показ toast

## 3. Core Modules

### Модуль: UI / Дизайн-система
- **Путь:** `index.html` → `<style>`
- **Ответственность:** Glass-морфизм, тёмная тема, CSS-переменные, анимации, адаптивность
- **Связи:** Используется всеми UI-компонентами через CSS-классы

### Модуль: Topbar + Settings
- **Путь:** `index.html` → `.topbar`, `.settings-panel`
- **Ответственность:** Логотип, бейдж версии, индикатор черновика, кнопка настроек, панель ввода вебхука
- **JS-функции:** `oS()`, `cS()`, `mWS()`, `sWH()`
- **Связи:** LocalStorage (`bx_webhook`)

### Модуль: Sidebar (Гайд)
- **Путь:** `index.html` → `aside.sidebar`
- **Ответственность:** Инструкция по заполнению формы, таблица маппинга полей → API
- **Связи:** Независим, скрывается на мобильных

### Модуль: Шаблоны задач
- **Путь:** `index.html` → секция шаблонов
- **Ответственность:** Предустановленные шаблоны описания задач (Bug, Feature, Hotfix и т.д.)
- **JS-функции:** `applyTemplate()`, `clearTemplate()`, `_lockScroll()`, `_unlockScroll()`
- **Связи:** Заполняют поля описания, скролл-лок для мобильных

### Модуль: Карточка 1 — Название
- **Путь:** `.card` Card 1
- **Ответственность:** Поле TITLE, огонёк важности (PRIORITY), кнопка «Повторить»
- **JS-функции:** `tF()` (toggle flame), `rL()` (repeat last), `val()` (валидация)
- **Связи:** LocalStorage (`bx_last_task`)

### Модуль: Карточка 2 — Команда
- **Путь:** `.card` Card 2
- **Ответственность:** Выбор Разработчика (RESPONSIBLE_ID), Постановщика (CREATED_BY), Наблюдателя, EOD-галка
- **JS-функции:** `renderChips()`, `sC()` (select chip)
- **Данные:** Жёстко закодированные массивы `DEVS`, `MGRS`, `OBSS` с Bitrix24 user ID
- **EOD:** Чекбокс «EOD» в заголовке карточки, выровнен вправо, включён по умолчанию
- **Связи:** API-запрос на добавление наблюдателя, EOD-комментарий от бота

### Модуль: Карточка 3 — Сроки
- **Путь:** `.card` Card 3
- **Ответственность:** Дата дедлайна + оценка времени в часах
- **Особенность:** Дедлайн всегда инициализируется текущей датой (локальное время, не UTC)
- **Оценка времени:** Поле ввода часов (целых или дробных, запятая/точка), отправляется как `TIME_ESTIMATE` (часы × 3600 сек) + `ALLOW_TIME_TRACKING: 'Y'`
- **Связи:** Поля DEADLINE и TIME_ESTIMATE в payload

### Модуль: Карточка 4 — Описание
- **Путь:** `.card` Card 4
- **Ответственность:** Структурированное описание задачи — секции:
  - Цель (textarea)
  - Что сделать (динамические поля, до 10)
  - Критерии приёмки (динамические поля, до 10)
  - Контакт (textarea)
  - Материалы (textarea)
  - Файлы (drag & drop + file input + clipboard paste Ctrl+V)
  - Предпросмотр описания (живая сборка)
- **JS-функции:** `bD()` (build description), `up()` (update preview), `ufi()`, `initDynamicFields()`, `addDynamicField()`, `removeDynamicField()`, `resetDynamicFields()`, `getDynamicValues()`, `ag()` (auto-grow textarea)
- **Связи:** `UF_TASK_WEBDAV_FILES`, `tasks.task.update`

### Модуль: Карточка 5 — Проект
- **Путь:** `.card` Card 5
- **Ответственность:** Выбор проекта (GROUP_ID)
- **JS-функции:** `renderProjs()`, `sP()`
- **Данные:** Жёстко закодированный массив `PROJS` с проектами и их Bitrix24 group ID

### Модуль: API-интеграция (создание задачи)
- **Путь:** `index.html` → `<script>` функция `handleSubmit()`
- **Ответственность:**
  - Создание задачи через `tasks.task.add`
  - Добавление наблюдателя через 4 варианта эндпоинта с фоллбэком
  - Добавление бота (154) как наблюдателя (для прав на EOD-комментарий)
  - EOD-комментарий через хук бота (154)
  - Загрузка файлов: FormData метод
  - Защита от двойного клика (`isSubmitting` flag)
  - Проверка на дубликат задачи (5 мин)
- **Связи:** Bitrix24 REST API (2 вебхука), LocalStorage

### Модуль: EOD-комментарий
- **Путь:** `index.html` → функция `addEodComment(hook, tid, projId)`
- **Ответственность:** Отправка шаблона EOD (End of Day) как комментарий к задаче от имени бота
- **Формат:** BBCode (`[b]`, `[i]`, не HTML)
- **Шаблон:**
  ```
  [i]Заполни EOD при срезе задачи, если она решается более 4х часов и по завершении задачи:[/i]
  [i]Стадия тестирования обязательна при завершении задачи[/i]
  [i]Не забываем прикладывать инструкцию по завершению задачи[/i]  ← только для проекта Dakota (ID=32)

  [b]Done[/b]:
  [b]Test[/b]:
  [b]Next[/b]:
  [b]Block[/b]:
  ```
- **Условие:** Бот (154) должен быть наблюдателем задачи, иначе нет прав на комментарий
- **Вебхук бота:** `https://1c-cms.bitrix24.ru/rest/154/f896em13hhazm006/`
- **Связи:** Вызывается из `handleSubmit()` если EOD-галка стоит

### Модуль: Загрузка файлов
- **Путь:** `index.html` → функция `uploadFiles(hook, tid)`
- **Ответственность:** Загрузка файлов на диск Bitrix24 и привязка к задаче
- **Метод:** Только FormData (единственный рабочий метод). Raw binary был убран в v7.7 — он «сжигал» upload URL токен
- **Имена clipboard-файлов:** Уникальные с timestamp (`clipboard-1716374400000.png`)
- **Связи:** `disk.folder.uploadfile`, `tasks.task.update`

### Модуль: Черновик + beforeunload
- **Путь:** `index.html` → `saveDraft()`, `clearDraft()`, `loadDraft()` (закомментирован вызов)
- **Ответственность:**
  - `beforeunload` предупреждение если форма заполнена
  - `saveDraft()` — автосохранение в LocalStorage
  - `loadDraft()` — восстановление (закомментировано в DOMContentLoaded)
  - Форма всегда сбрасывается при перезагрузке страницы
- **Связи:** LocalStorage

### Модуль: История задач
- **Путь:** `index.html` → `pushHistory()`, рендер истории
- **Ответственность:** Отображение недавно созданных задач (ID, название, разработчик, проект)
- **Связи:** LocalStorage

### Модуль: Toast-уведомления
- **Путь:** `index.html` → `.toast` + `showToast()`
- **Ответственность:** Success/error/warn уведомления, автоскрытие через 6 секунд
- **Связи:** Вызывается из всех модулей

## 6. Known Issues & Problems

- **CORS-ограничения:** Прямые `fetch()` к Bitrix24 API из `file://` не работают. Требуется запуск через локальный сервер или GitHub Pages
- **Наблюдатель — нет единого рабочего эндпоинта:** `task.observers.add` и `tasks.task.observers.add` возвращают 404. Фоллбэк через `tasks.task.update` с `AUDITORS` работает. Первые 2 попытки — косметический шум в консоли
- **Жёстко закодированные ID:** Массивы `DEVS`, `MGRS`, `OBSS`, `PROJS` содержат конкретные Bitrix24 user/group ID. При изменении состава команды/проектов — правка кода

## 10. Decisions & Reasoning

- **Vanilla JS без фреймворков:** Форма — единственная страница, состояние минимально. Нулевой бандл, мгновенная загрузка
- **Один HTML-файл:** Удобно деплоить — загрузил один файл на GitHub Pages. Нет сборки, нет зависимостей
- **Два вебхука:** EOD должен приходить от бота, а не от пользователя. Bitrix24 привязывает комментарии к владельцу вебхука, AUTHOR_ID не работает. Единственное решение — отдельный вебхук от бота
- **Бот как наблюдатель:** Bitrix24 не позволяет комментировать чужие задачи без участия. Бот добавляется наблюдателем автоматически перед EOD-комментарием
- **BBCode вместо HTML:** Bitrix24 комментарии рендерят BBCode, HTML-теги отображаются как текст
- **FormData только для файлов:** Raw binary «сжигал» upload URL токен. После ошибки токен нельзя переиспользовать
- **Дедлайн локальное время:** `toISOString()` даёт UTC — может быть другой день. Используем `getFullYear/getMonth/getDate`
- **Форма сбрасывается при перезагрузке:** loadDraft() закомментирован — форма всегда чистая. Черновик сохраняется только для beforeunload предупреждения
- **Скролл-лок через position:fixed:** Единственный надёжный способ предотвратить прыжок на мобильных при autoGrow текстареей
- **Жёстко закодированные ID:** Команда и проекты стабильны. Нет смысла тянуть справочники из API
- **Хук бота захардкожен:** Не настраивается через UI — проще и безопаснее. При перевыпуске — правка одной строки в коде
- **Структурированное описание:** Задачи в команде были неструктурированными. Форма заставляет заполнить цель, что сделать, критерии приёмки

## 11. Limitations / Tech Debt

- **Мёртвый код:** Переменная `tags`, функции `addTag()`, `removeTag()`, `rT()` — UI удалён, код остался. В payload: `TAGS:tags` (пустой массив)
- **Один монолитный файл:** ~1682 строки HTML+CSS+JS. Нет модульности, нет тестов
- **Минифицированные имена функций:** `ag()`, `ufi()`, `up()`, `sC()`, `sP()`, `rL()`, `hS()`, `hF()`, `rF()`, `bD()`, `oS()`, `cS()`, `mWS()`, `sWH()`, `uRB()`, `fSz()`, `tF()`, `esc()`, `showT()`. Нечитаемо без документации
- **Нет unit-тестов:** Нет тестирования, ручная проверка
- **Нет CI/CD:** Коммиты напрямую в main, GitHub Pages Auto-deploy
- **CORS-зависимость:** Работает только через HTTP-сервер, не из `file://`

## 13. Quick Context for AI

### Как писать код в этом проекте
- Всё в одном файле `index.html`: CSS в `<style>`, JS в `<script>`, HTML в `<body>`
- Нет сборки, нет npm, нет бандлера. Прямой редактируемый HTML
- Деплой через push в main → GitHub Pages Auto-deploy
- Не добавляй внешние зависимости без крайней необходимости

### Принципы
- Минимализм: минимум зависимостей, нулевой бандл
- Практичность над красотой: форма должна работать надёжно
- Фоллбэки: если что-то может не работать — предусмотрены варианты
- Bitrix24 API нестабилен: всегда перебирай несколько вариантов эндпоинта
- BBCode для Bitrix24 комментариев, НЕ HTML

### Стиль кода
- CSS: Custom Properties (`--accent`, `--bg`, и т.д.), краткие имена классов, glass-морфизм
- JS: ES6+, async/await, краткие/минифицированные имена функций (исторически сложилось)
- HTML: Semantic5 где возможно, но в основном div-ная вёрстка с классами
- Форматирование: компактное, без лишних пробелов (исторически)
- Язык UI и комментариев: русский

### Ограничения
- Bitrix24 REST API через входящий вебхук — ограниченный набор методов, нет WebSocket, нет подписок
- Файлы загружаются только через `disk.folder.uploadfile` в папку id=3, затем привязываются к задаче
- Наблюдатель — только через `tasks.task.update` с AUDITORS после создания задачи
- Комментарии от бота — только если бот является наблюдателем задачи
- AUTHOR_ID не работает для вебхуков — комментарии всегда от владельца вебхука
- Мобильная версия: sidebar скрыт, сетка в одну колонку
