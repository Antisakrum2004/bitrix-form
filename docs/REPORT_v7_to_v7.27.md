## Что добавили — подробно

### 1. AI-ассистент в форме (v7.16–v7.18)

В форму добавлена кнопка AI. Пользователь описывает задачу свободным текстом или голосом (через Web Speech API, работает в Chrome/Edge). AI через OpenRouter (модели deepseek/gemini/gpt-4o-mini/claude) разбирает текст и автоматически заполняет:
- Заголовок
- Цель (что нужно сделать)
- Описание (с BBCode-форматированием)
- Критерии приёмки
- Разработчика (подбирает по контексту задачи)
- Проект (подбирает по контексту)
- Дедлайн (оценивает по сложности)

Достаточно проверить, поправить при необходимости и нажать «Создать». Экономит 3–5 минут на каждой задаче.

### 2. Семантический поиск похожих задач (v7.26, v7.27)

**Проблема, которую решали:**  
Раньше поиск похожих задач в Bitrix24 работал только по точному совпадению слов. Если менеджер писал «отрицательные остатки», система не находила задачу «Проверить почему пишет минус резерв по остатки» — хотя по смыслу это одно и то же. В итоге создавались дубликаты.

**Что сделали:**  
Внедрили векторный поиск через Supabase pgvector. Каждая задача получает embedding (числовое представление смысла) через OpenAI модель `text-embedding-3-small`. При поиске система сравнивает не слова, а смыслы — находит задачи даже если они написаны другими словами.

**Результат:**  
- 1201 задача за 2026 год проиндексирована
- Запрос «отрицательные остатки» теперь находит 5 релевантных задач, включая #7646 «минус резерв», которую раньше не находил
- Стоимость индексации: ~$0.005 (5 центов за всё)
- Стоимость каждого поиска: ~$0.00001

### 3. Гибридный поиск (v7.27)

Векторный поиск хорош для семантики, но слаб для точных совпадений (артикулы, номера документов). Добавили гибридный скоринг: 70% — семантика, 30% — точное совпадение слов через pg_trgm (Postgres extension для fuzzy matching).

**Пример улучшения:**  
Запрос «настроить права доступа пользователям»:
- В чисто векторном: top-1 «Настроить доступы пользователей» (sim=0.702)
- В гибридном: тот же top-1, но с учётом точного совпадения слов sim=0.684 — система «увереннее», что это релевантно

### 4. Разделение UI: Дубликаты / Похожие (v7.27)

В AI-панели список «Похожие задачи» разбит на 2 блока:

- **Возможные дубликаты** (similarity ≥ 85%) — вероятно, задача уже создана. Жёлтая подсветка, предупреждение. Если таких нет — показываем зелёную плашку «Дубликатов не найдено — можно создавать спокойно».
- **Похожие задачи** (similarity 40–85%) — можно подсмотреть решение или обсудить с коллегой, у которого похожая задача. Синяя подсветка.

У каждой задачи видно процент совпадения, ответственного, статус и проект.

### 5. Поле «Ключевые слова» для управления поиском (v7.25)

Если AI не нашёл похожих задач или менеджер хочет уточнить поиск — добавлено поле «Ключевые слова». Вводишь любые слова или фразы (через запятую, пробел или в кавычках) — они участвуют в поиске аналогичных задач.

Например, AI заполнил форму по тексту «оптимизировать выгрузку», но похожих не нашлось. Менеджер добавляет ключевые слова `озон, яндекс.маркет, выгрузка товаров` — и поиск находит 3 релевантные задачи.

### 6. Автоматическое обновление индекса (v7.27)

Настроен ежедневный запуск в 06:00 МСК (через GitHub Actions):
1. Скрипт скачивает все задачи из Bitrix24, созданные с 2026-01-01
2. Для новых/изменённых задач получает embedding через OpenRouter
3. Записывает в Supabase pgvector

Это гарантирует, что поиск всегда актуален — новые задачи, созданные за день, будут в индексе на следующее утро. Стоимость ~$0.005/день (~$1.8/год).

### 7. EOD-инспектор и напоминания (отдельный модуль, v2–v5)

Параллельно с формой задач развивался бот «EOD-инспектор»:
- В 20:00 МСК проверяет у кого из разработчиков нет отчёта о проделанной работе за день
- Отправляет личное сообщение руководителю со списком
- В 18:00 и 19:00 МСК — мягкие напоминания разработчикам, у которых ещё нет EOD
- Раз в неделю — график количества задач в работе по каждому разработчику

## Технические детали (для IT-отдела)

**Архитектура:**
- Frontend: `index.html` (один файл, GitHub Pages) — vanilla JS, без сборки
- Backend: Next.js 16 на Vercel — API routes для AI-операций
- Векторная БД: Supabase pgvector (Free тариф, 500 MB, занято ~12 MB)
- AI-провайдер: OpenRouter (прокси к OpenAI/Anthropic/Google и др.)
- Автообновление: GitHub Actions (cron 03:00 UTC)

**Репозитории:**
- `Antisakrum2004/bitrix-form` — фронтенд + EOD-инспектор + скрипты синхронизации
- `Antisakrum2004/bitrix-form-AI` — backend на Vercel

**Стоимость эксплуатации:**
- Supabase: $0 (Free тариф, запас ×40)
- Vercel: $0 (Hobby, запас огромный)
- GitHub Actions: $0 (2000 мин/мес, крон жрёт ~15 мин/мес)
- OpenRouter (AI): ~$1-3/мес в зависимости от активности
- GitHub Pages: $0

**Точки отката:**
- `v7.26.2-rollback-2026-06-30` — к версии до гибридного поиска/UI-разделения/крона (семантический поиск остаётся)
- `v7.25-rollback-2026-06-30` — к версии до семантического поиска (возврат к лексическому)

## Что НЕ сделано (отложено)

- **AI re-rank топ-5** — переоценка найденных задач через LLM для большей точности. Отложено: добавит 300-800мс задержки и ~$0.0005/запрос. Текущего гибрида достаточно.
- **Индексация задач 2025 года и ранее** — по запросу, расширение `SINCE_DATE` в скрипте.
- **Webhook от Bitrix24** для мгновенной индексации новых задач — вместо крона. Сложнее в настройке, реальной потребности пока нет.

## Как использовать новые возможности

**Создать задачу через AI:**
1. Открыть https://antisakrum2004.github.io/bitrix-form/
2. Нажать оранжевую кнопку AI справа внизу
3. Описать задачу текстом (или голосом — иконка микрофона в Chrome)
4. Проверить заполненные поля
5. При необходимости — добавить ключевые слова для уточнения поиска похожих
6. Нажать «Создать задачу»

**Проверить похожие задачи:**  
AI автоматически ищет похожие при заполнении формы. Смотреть блоки:
- «Возможные дубликаты» (жёлтый) — вероятно, задача уже есть
- «Похожие задачи» (синий) — для контекста и подсмотра решений

**Проверить работу поиска вручную:**  
```
curl -X POST https://bitrix-form-ai.vercel.app/api/ai-similar \
  -H "Content-Type: application/json" \
  -d '{"text":"минус резерв","threshold":0.4,"limit":5}'
```
